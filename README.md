# 阿剛老師的剛好系列：課堂互動程式整合大禮包

這個資料夾是一套可直接用瀏覽器開啟或部署到靜態網站的課堂互動工具包，整合「剛好學」、「剛好測」、「剛好白板」、「剛好簡報」與 Firebase 設定工具。整套工具以 HTML 單頁應用為主，不需要後端伺服器；師生即時同步、題庫、白板、簡報互動資料主要透過 Firebase Firestore 儲存與同步。

適合用在實體課、線上同步課、分組討論、即時測驗、閱讀理解、互動簡報、協作白板與學生作品收集。

## 檔案結構

| 檔案 | 用途 |
| --- | --- |
| `index.html` | 整合入口首頁，提供四個主工具的入口與 Firebase 通用設定教學。 |
| `01classtools.html` | 「剛好學」課堂互動工具，包含多種即時互動模式。 |
| `02classwhiteboard.html` | 「剛好白板」即時互動數位白板，支援多人協作、PDF、圖片、便利貼與匯出。 |
| `03classppt.html` | 「剛好簡報」即時同步簡報工具，支援 PDF 轉簡報、學生互動、畫記與回應收集。 |
| `04classquiz.html` | 「剛好測」AI 題庫與測驗系統，支援題庫、學生作答、成績分析與匯出。 |
| `firebase_setup_tools.html` | 舊版 Firebase 設定更新器，可將 Firebase Config 寫入指定 HTML 後下載。 |
| `hero-image.jpg` | 首頁使用的介紹圖片。 |
| `README.md` | 本說明文件。 |

## 系統特色

- 純前端靜態網站：可直接開啟 HTML，或部署到 GitHub Pages、EzPage、Netlify 等靜態空間。
- Firebase 即時同步：透過 Firestore 同步學生名單、作答、白板物件、簡報狀態與測驗資料。
- 教師與學生分流：多數工具都有教師入口與學生入口，方便課堂操作。
- 支援 QR Code 與專用連結：設定 Firebase 後可產生含設定資訊的網址與 QR Code，讓其他裝置免重新設定。
- 支援 localStorage：Firebase Config、AI API Key 等設定可保存在目前瀏覽器。
- 可備份與匯出：白板、簡報、題庫與作答結果可依工具功能匯出為 JSON、CSV、PDF 或媒體檔。
- CDN 載入函式庫：使用 Tailwind CSS、Font Awesome、Firebase SDK、PDF.js、QRCode.js、Vue、Chart.js、Fabric.js、D3 等前端函式庫。

> 注意：由於多數函式庫與 Firebase SDK 透過 CDN 載入，正式上課時建議使用可連外網的環境。

## 快速開始

1. 開啟 `index.html`。
2. 點選需要的工具：
   - 剛好學：`01classtools.html`
   - 剛好測：`04classquiz.html`
   - 剛好白板：`02classwhiteboard.html`
   - 剛好簡報：`03classppt.html`
3. 第一次使用請先設定 Firebase。
4. 教師端建立活動、白板、簡報或測驗。
5. 將學生連結或 QR Code 提供給學生。

若只是本機測試，可直接雙擊 HTML 開啟；若要讓學生從不同裝置連線，請將整個資料夾部署到可公開存取的 HTTPS 網址。

### 一般教師使用 GitHub 網頁版

一般教師不一定需要自行下載、修改或重新部署 HTML。可直接使用作者已發布在 GitHub Pages 的工具網頁，進入需要的課堂工具後，在工具網頁介面的「Firebase 設定」中貼上自己的 Firebase 六行設定值即可開始使用。

可使用的 GitHub Pages 工具網址：

| 工具 | GitHub Pages 網址 |
| --- | --- |
| 剛好學 | <https://github.com/kentxchang/kentclasstool2/01classtools.html> |
| 剛好白板 | <https://github.com/kentxchang/kentclasstool2/02classwhiteboard.html> |
| 剛好簡報 | <https://github.com/kentxchang/kentclasstool2/03classppt.html> |
| 剛好測 | <https://github.com/kentxchang/kentclasstool2/04classquiz.html> |

建議課堂使用流程：

1. 教師開啟對應的 GitHub Pages 工具網址。
2. 在工具頁面的「Firebase 設定」中，貼上自己的 Firebase Config 六行設定值。
3. 套用設定後，由網頁介面產生學生專用連結。
4. 將學生專用連結製作成 QR Code，或直接使用工具內建產生的 QR Code。
5. 學生用平板掃描 QR Code 連入一次後，Firebase 設定值會自動保存在該平板瀏覽器的本地暫存中。
6. 之後同一批學生再上課時，可直接從原始 GitHub Pages 工具網址進入，通常不必重新掃描含 Firebase 設定值的專用連結。

這個機制的重點是：第一次連入「含 Firebase 設定值的學生專用連結」後，學生平板瀏覽器會記住該老師的資料庫設定；下次只要沒有清除瀏覽器資料，就能直接使用原始工具網址與同一位老師的 Firebase 專案互動。

## Firebase 設定總覽

本工具包的即時互動功能需要 Firebase Firestore 與匿名登入。建議建立兩個 Firebase 專案：

| Firebase 專案 | 建議用途 | 對應工具 |
| --- | --- | --- |
| 通用專案 | 課堂短期互動、白板、簡報同步 | 剛好學、剛好白板、剛好簡報 |
| 剛好測專案 | 題庫、成績、測驗資料長期保存 | 剛好測 |

基本設定流程：

1. 前往 Firebase Console 建立專案。
2. 建立 Web App，取得 `firebaseConfig`。
3. 啟用 Authentication 的匿名登入。
4. 建立 Firestore Database。
5. 在 Firestore Rules 貼上工具提供的規則。
6. 在各工具右上角或設定視窗貼入 `firebaseConfig`。
7. 產生教師或學生專用連結與 QR Code。

`index.html` 內建 Firebase 圖文設定教學。多數主工具也有自己的 Firebase 設定視窗。

### Firebase Config 貼上格式

通常只需要貼上 `firebaseConfig` 大括號中的六行內容：

```js
apiKey: "AIza...",
authDomain: "your-project.firebaseapp.com",
projectId: "your-project-id",
storageBucket: "your-project.appspot.com",
messagingSenderId: "123456789",
appId: "1:123456789:web:abcdef"
```

### 設定來源優先順序

各工具略有差異，但大致採用以下優先順序：

1. URL 參數中的 Firebase 設定。
2. 瀏覽器 `localStorage` 中的 Firebase 設定。
3. HTML 原始碼內的備用設定。
4. 未設定時顯示提示或停留在設定狀態。

### 安全性提醒

目前工具頁提供的 Firestore 規則偏向教學便利，允許匿名使用者讀寫指定資料。這適合課堂互動，但不適合存放敏感資料。若要長期公開部署，請至少注意：

- 不要保存個資、成績以外的敏感資訊或機密內容。
- 剛好測建議使用獨立 Firebase 專案，避免題庫與其他互動資料混在一起。
- 定期清理不需要的白板、簡報與課堂資料。
- 若學校或單位有資訊安全要求，請自行收斂 Firestore Rules。

## 各工具說明

### 1. 剛好學：課堂互動 so easy

檔案：`01classtools.html`

定位：全能型課堂互動中心，教師建立教室代碼，學生輸入教室代碼與姓名加入。適合隨堂提問、作品收集、投票、閱讀測驗、搶答與分組活動。

主要流程：

1. 教師點選「我是教師」。
2. 輸入或選擇教室代碼。
3. 選擇互動模式。
4. 學生點選「我是學生」，輸入教室代碼與姓名。
5. 教師即時查看學生作答、統計、未作答名單與作品。

互動模式：

| 模式 | 說明 |
| --- | --- |
| 是非題 | 學生快速選擇是/否，教師端即時統計。 |
| 選擇題 | 可用預設 1-4 選項，也可建立自訂題目。 |
| 排序題 | 學生拖曳項目排序，適合流程、時間線與步驟題。 |
| 配對題 | 學生將左右項目配對，適合詞義、概念、人物與事件。 |
| 文字題 | 學生送出文字回應，教師可查看、下載與進行 AI 分析。 |
| 即時文字雲 | 學生輸入短詞，教師端即時產生文字雲。 |
| 繪圖題 | 學生在畫布上作答，可搭配教師上傳背景圖片。 |
| 派送網址/HTML | 教師派送網址、YouTube 或 HTML 內容給學生端開啟。 |
| 錄音題 | 學生使用麥克風錄音並送出。 |
| 閱讀測驗 | 顯示閱讀文本、題目與畫記工具，學生可閱讀後作答。 |
| 便利貼牆 | 學生送出文字或圖片便利貼，教師端集中展示。 |
| 專注力遊戲 | 學生完成數字點選挑戰，教師端依完成時間分組顯示。 |
| 搶答 | 學生點擊搶答按鈕，教師端顯示搶答結果。 |
| 快速投票 | 教師自訂問題與選項，學生投票後即時統計。 |

教師端功能：

- 顯示在線學生人數。
- 暫停或恢復互動。
- 抽籤學生。
- 查看未作答名單。
- 查看統計圖表。
- 下載作答結果。
- 啟動同儕互評。
- 查看題目與正確答案。
- AI 文字分析與 AI 出題設定。
- 背景圖片上傳與貼上。
- 產生 Firebase 專用網址與 QR Code。

AI 相關：

- 可設定 Gemini API Key。
- 可選擇 Gemini 2.5 Flash、Gemini 2.5 Flash Preview、Gemma 3 27B IT 等模型選項。
- 選擇題可用 AI 依主題產生題目。
- 文字題可進行 AI 摘要與熱門詞彙分析。

資料與備份：

- 作答與互動資料寫入 Firestore。
- 教師端可下載學生回應或媒體資料。
- 瀏覽器會保存部分本機設定，如 Firebase Config、AI Key 與近期教室資訊。

### 2. 剛好測

檔案：`04classquiz.html`

定位：AI 輔助題庫與測驗系統。教師建立題庫、啟用測驗，學生輸入姓名後作答。系統會保存作答紀錄、計算分數並提供分析報告。

教師密碼：

- 預設教師密碼為 `tpet`。

主要流程：

1. 教師進入「我是老師」。
2. 輸入教師密碼。
3. 在系統設定中貼上剛好測專用 Firebase Config。
4. 視需要設定 Gemini API Key 與 AI 模型。
5. 新增題庫或編輯既有題庫。
6. 啟用某一份測驗。
7. 學生進入學生端，輸入姓名後作答。
8. 教師查看成績分佈、題目分析與學生作答詳情。

支援題型：

| 題型 | 說明 |
| --- | --- |
| 一般選擇題 | 支援 AI 自動出題、手動輸入、圖片/PDF 來源、解析與難度。 |
| 閱讀測驗 | 可貼上文本、上傳文字檔，或由 AI 依主題產生文本與題目。支援 PIRLS 四層次設定。 |
| 素養題組 | 支援情境文本、圖片、PDF、文字來源與 AI 產生題組。 |
| 配對題 | 支援 AI 生成配對、手動輸入配對，學生端以拖曳方式完成。 |
| YouTube 影片題 | 教師貼上 YouTube 網址與字幕/逐字稿，AI 可依字幕產生題目。 |

題庫功能：

- 題庫分類與搜尋。
- 新增、編輯、刪除、複製題庫。
- 啟用或結束測驗。
- 題目可手動增修。
- AI 產題後仍可逐題檢查與修改。
- 偵測解析與答案可能不一致的題目。

匯出格式：

- 一般選擇題可匯出 Wordwall、Kahoot!、書商格式、Wayground 格式。
- 配對題可匯出 A 欄在前或 B 欄在前的 CSV 格式。

分析功能：

- 成績分佈圖。
- 學生列表與個別作答詳情。
- 題目正確率分析。
- 配對題每組配對正確率。
- 閱讀測驗與素養題組可查看文本、學生畫記與答題細節。

設定與分享：

- 系統設定中可保存 Firebase Config。
- 可產生教師用完整功能連結與學生專用簡化連結。
- 學生專用連結會限制學生只看到學生登入介面。

### 3. 剛好白板：即時互動數位白板

檔案：`02classwhiteboard.html`

定位：多人同步白板。教師建立或接管白板 ID，學生輸入姓名與白板 ID 加入。適合共同筆記、分組討論、資料標註、腦力激盪與作品展示。

教師密碼：

- 預設教師密碼為 `tpet`。

主要流程：

1. 教師輸入密碼。
2. 選擇既有白板或輸入新白板 ID。
3. 學生輸入姓名與白板 ID 加入。
4. 師生在同一白板上即時操作。
5. 教師可匯出 PDF、匯出 JSON 或匯入 JSON。

白板工具：

| 工具 | 說明 |
| --- | --- |
| 移動畫布 | 拖曳畫布視角。 |
| 選取/移動 | 選取、移動與操作物件。 |
| 畫筆 | 支援顏色與筆寬。 |
| 圖形 | 支援箭頭、圓形、矩形。 |
| 便利貼 | 可新增不同顏色便利貼。 |
| 插入圖片 | 上傳圖片到白板。 |
| 插入 PDF | 選取 PDF 頁面後插入白板。 |
| 刪除 | 刪除選取物件。 |
| 縮放 | 支援縮放滑桿。 |

教師專用功能：

- 鎖定或解鎖學生工具。
- 開放或限制學生編輯所有物件。
- 顯示或隱藏作者姓名。
- 查看線上參與者。
- 切換白板。
- 刪除單一白板或清空整個資料庫。
- 匯出 PDF。
- 匯出 JSON 備份。
- 匯入 JSON 還原。
- 產生 Firebase 專用網址、短網址與 QR Code。

資料保存：

- 白板資料會寫入 Firestore。
- 離開白板不會刪除資料。
- 刪除白板或清空資料庫會永久移除資料，操作前請先匯出 JSON。

### 4. 剛好簡報：即時簡報互動工具

檔案：`03classppt.html`

定位：將 PDF 簡報轉成可同步播放的互動簡報。教師控制投影片進度，學生端同步觀看，並可依投影片模式送出塗鴉、文字或選擇題答案。

教師密碼：

- 預設教師密碼為 `tpet`。

主要流程：

1. 教師先在右上角設定 Firebase。
2. 進入教師入口並輸入密碼。
3. 上傳 PDF 簡報，或匯入先前匯出的簡報 JSON。
4. 系統將 PDF 頁面轉成投影片圖片。
5. 教師設定每頁互動模式。
6. 產生學生連結或 QR Code。
7. 開始同步簡報。
8. 學生輸入姓名加入課堂。
9. 教師可查看、下載、比較或重置學生回應。

投影片模式：

| 模式 | 說明 |
| --- | --- |
| 簡報 | 一般同步播放模式。 |
| 塗鴉 | 學生可在投影片上畫記並送出作品。 |
| 文字 | 學生可送出文字回應。 |
| 選擇 | 學生作答 1-4 或 A-D 選項，可設定正確答案。 |

教師控制功能：

- 開始或停止同步。
- 上一頁、下一頁與縮圖導航。
- 插入新頁面，可建立純色頁或圖片頁。
- 設定每頁網址，學生端可看到連結圖示。
- 畫筆、橡皮擦、清除本頁畫記。
- 雷射筆同步。
- 顯示在線學生列表。
- 查看學生回應。
- 下載全部回應。
- 最多選擇 4 件作品進行比較。
- 重置所有回應。
- 匯出簡報 JSON。
- 結束並刪除簡報資料。

注意事項：

- PDF 會轉成圖片後上傳到 Firestore，單頁圖片過大可能上傳失敗。
- 建議先壓縮大型 PDF 或拆成較小檔案。
- 完成簡報後可匯出 JSON 保存，下次用「匯入簡報 JSON」恢復。
- 課程結束後建議使用「結束簡報」清理雲端資料。

### 5. Firebase 設定更新器

檔案：`firebase_setup_tools.html`

定位：舊版設定工具，用於將 Firebase Config 寫入 HTML 後下載成已設定版本。新版流程建議優先使用各主工具內建的「Firebase 設定」按鈕；此檔案保留給需要產生固定 HTML 檔的人使用。

使用方式：

1. 開啟 `firebase_setup_tools.html`。
2. 上傳本機 HTML，或從選單選擇工具範本。
3. 貼上 Firebase Config。
4. 點選「下載設定好的 HTML」。
5. 取得檔名含 `_已設定.html` 的新檔案。

支援範本：

- 剛好學
- 剛好白板
- 剛好簡報
- 剛好測

限制：

- 依 HTML 中既有的 Firebase 設定區塊進行取代。
- 若 HTML 結構大幅調整，可能無法找到可取代的設定區塊。
- 現在多數主工具已支援直接貼上設定與產生專用網址，通常不必使用此工具。

## 建議部署方式

### 本機使用

適合教師自己測試功能：

1. 直接開啟 `index.html`。
2. 設定 Firebase。
3. 同一台電腦測試教師與學生流程。

限制：如果學生要用自己的裝置加入，必須能連到這些 HTML 所在位置，因此本機檔案通常不適合正式課堂分享。

### 靜態網站部署

適合正式上課：

1. 將整個資料夾上傳到 GitHub Pages、EzPage、Netlify、Cloudflare Pages 或其他靜態網站服務。
2. 確認 `index.html`、四個主工具 HTML、`firebase_setup_tools.html` 與 `hero-image.jpg` 都在同一層。
3. 使用 HTTPS 網址開啟。
4. 在每個工具中貼上 Firebase Config。
5. 產生教師/學生連結與 QR Code。

不建議只上傳單一 HTML，因為首頁圖片與工具互相連結可能失效。若只要使用某一個工具，至少要確認該工具所有外部 CDN 可正常載入。

## 常見問題

### 學生端無法同步或看不到活動

- 檢查教師端與學生端是否使用同一個 Firebase 專案設定。
- 檢查學生是否輸入正確教室代碼、白板 ID 或使用正確學生連結。
- 檢查 Firebase Authentication 是否已啟用匿名登入。
- 檢查 Firestore Database 是否已建立。
- 檢查 Firestore Rules 是否已發布。
- 檢查瀏覽器主控台是否有 Firebase 初始化錯誤。

### Firebase 設定貼上後仍失敗

- 只貼上大括號內的六行，不要重複貼 `const firebaseConfig = {` 與 `};`，除非工具提示允許完整貼上。
- 確認 `apiKey`、`authDomain`、`projectId`、`storageBucket`、`messagingSenderId`、`appId` 都存在。
- 若使用 URL 專用連結後仍失敗，請清除本地 Firebase 設定並重新整理。
- 剛好測建議使用獨立 Firebase 專案，不要混用通用專案。

### 學生平板一直無法跟老師連線

若學生曾經用同一台平板連過其他老師的課堂工具，瀏覽器本地暫存中可能仍保存前一位老師的 Firebase 資料庫設定。這會造成學生雖然開啟了工具頁，卻連到錯誤的資料庫，因此看不到目前老師的活動。

處理方式：

1. 請學生在該工具頁面打開「設定」或「Firebase 設定」。
2. 點選「清除本地 Firebase 設定」、「清除所有設定並登出」或類似的清除本地暫存按鈕。
3. 重新整理頁面。
4. 請學生重新掃描老師提供的「含 Firebase 設定值」學生專用 QR Code 或連結。
5. 重新連入後，新的 Firebase 設定會再次保存到學生平板瀏覽器中。

若清除工具內設定後仍無法連線，可再清除該瀏覽器對此網站的網站資料，或改用無痕視窗測試是否為暫存設定造成。

### QR Code 或短網址無法生成

- 檢查網路是否可連外。
- 短網址服務可能暫時不可用，可改用原始網址。
- 若用 `file://` 本機開啟，部分工具只支援 HTTP/HTTPS 產生可分享短網址。

### 剛好簡報上傳 PDF 失敗

- PDF 頁數或單頁圖片可能太大。
- 先壓縮 PDF 或拆成多個檔案。
- 避免使用超高解析度圖片型 PDF。
- 可先用少頁數 PDF 測試 Firebase 與同步功能是否正常。

### 錄音題無法錄音

- 確認瀏覽器允許麥克風。
- 建議使用 HTTPS 網址，部分瀏覽器會限制 `file://` 或非 HTTPS 的錄音權限。
- 檢查學生裝置是否有可用麥克風。

### AI 功能無法使用

- 檢查 Gemini API Key 是否已儲存。
- 檢查選擇的模型是否可用。
- 檢查來源內容是否過長或格式不完整。
- 若是 PDF 或圖片出題，先確認內容已成功解析或載入。

## 教學使用建議

- 上課前先建立 Firebase 專案並測試每個工具。
- 剛好學、白板、簡報可共用通用 Firebase 專案；剛好測另建專案。
- 每堂課使用固定教室代碼或白板 ID，方便學生加入。
- 使用 QR Code 時，建議同時提供原始網址，避免短網址服務不穩。
- 高風險操作前先匯出備份，例如刪除白板、結束簡報、清空資料庫。
- 測驗題庫正式使用前，先用教師帳號預覽並檢查答案與解析。
- AI 生成題目後一定要人工審題，避免答案錯誤或難度不符合學生程度。
